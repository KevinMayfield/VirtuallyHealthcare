Behavior: Capability Statements

The following artifacts define the specific capabilities that different types of systems are expected to have in order to comply with this implementation guide. Systems conforming to this implementation guide are expected to declare conformance to one or more of the following capability statements.

Document Index Provider based on IHE MHD

Provider supports read-only access to a patient document and clinical index for the purposes of direct care and subject of care access. Backwards compatible with IHE XDS.
Patient Index Provider based on IHE PDQm

Provider supports read-only access to a patient index (PAS, MPI or PDS) for the purposes of direct care and subject of care access.
Query API Provider based on IHE QEDm and IPA

Provider supports read-only access structured clinical record held within EHR’s.
Scheduling Provider

Scheduling API blueprint for appointment providers (PAS and EPR) based on

  • NHS Booking - FHIR API
  • GP Connect Appointment Management - FHIR API
  • IM1 supplier APIs
Structured Data Capture Provider

Providers supports read only access to sources of structured data capture (e.g. openEHR). Is currently focused on Finding Questionnaires (Form Definitions) and Finding Completed Questionnaires. Form Pre-Population is suppported by capabilities in other provider API’s.

  • FHIR Structured Data Capture
Workflow Provider

Is primarily focused on supporting clinical process. Technically this is a replacment of messaging with conversations and the workflow API is based on

  • FHIR Workflow
